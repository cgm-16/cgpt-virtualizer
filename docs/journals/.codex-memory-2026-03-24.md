## 시각 회귀 테스트 파이프라인

- Ori는 Playwright로 크롬 확장 기능을 테스트하는 현재 방식이 가능한지 확인했고, 확장 로드 방식은 Playwright 문서 기준으로 `chromium` persistent context를 계속 쓰는 방향으로 정리됐다.
- 다음 작업 목표는 Playwright 통합 테스트에 스크린샷 기반 시각 회귀 검증을 추가하고, CI에서 결과 이미지를 확인할 수 있게 만드는 것이다.
- 시각 정책은 골든 베이스라인 비교를 기본으로 하고, 초기 범위는 팝업과 대표 transcript 페이지 1개다.

## 저장소 상태

- 현재 저장소는 `main` 브랜치이고, `.worktrees/`, `.pnpm-store/`, `QA.md`가 untracked 상태다.
- 기존 Playwright 확장 픽스처는 `tests/integration/extension-fixture.ts`에서 `chromium.launchPersistentContext`와 `--disable-extensions-except`, `--load-extension` 플래그를 사용한다.
- 기존 GitHub Actions 워크플로는 `.github/workflows/playwright.yml` 하나만 있고 HTML 리포트만 아티팩트로 업로드한다.
