## Additional findings

- Chrome unpacked-extension load failure investigation:
  - `dist/content.js` currently starts with a top-level static `import` from `./assets/messages-*.js`.
  - `public/manifest.json` registers `content.js` under `content_scripts.js`, which loads it as a classic content script, not as a module.
  - `vite.config.ts` forces Rollup output `format: "es"` for all entry points, so shared code is emitted as ESM chunks and the content entry is not self-contained.
  - `tests/integration/content-bootstrap.spec.ts` loads `/dist/content.js` with `<script type="module">`, which masks the real Chrome content-script loading mode and would not catch this error.
