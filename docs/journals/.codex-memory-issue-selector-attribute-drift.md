## 배경

V1은 exact selector가 깨지면 fail-closed로 `Unavailable`이 되어야 합니다. 현재 mid-session selector failure 감시는 child removal/insertion에는 반응하지만, attribute/class만 바뀌는 drift에는 반응하지 않습니다.

## 근거

- `whitepaper.md:199-204`는 selector failure 시 `Unavailable + inert` 전략을 요구합니다.
- `src/content/failure.ts:21-55`의 `createSelectorFailureObserverManager()`는 `childList: true`, `subtree: true`만 관찰합니다.
- 같은 파일의 `sync()`는 observer callback이 실행될 때만 `resolveSelectors()`를 다시 확인합니다.
- 즉 transcript root나 scroll container가 DOM에 남아 있으면서 class/data attribute만 바뀌어 selector에서 이탈하면 `sync()`가 자동 호출되지 않습니다.
- 현재 통합 테스트는 fixture 교체로 인한 selector disappearance만 검증합니다. `tests/integration/content-bootstrap.spec.ts:236-311`

## 문제

실제 ChatGPT DOM drift가 node removal이 아니라 class/data attribute 변경 형태로 오면, 런타임은 더 이상 유효하지 않은 selector 계약 위에서 계속 동작할 수 있습니다. 이 경우 whitepaper의 fail-closed 전략이 깨집니다.

## 기대 동작

- mid-session selector failure 감시는 attribute/class 기반 drift도 감지해야 합니다.
- selector 계약이 깨지면 즉시 `Unavailable`을 보고하고 inert 상태로 전환해야 합니다.

## 완료 조건

- selector failure observer가 attribute drift를 감지합니다.
- attribute-only drift 회귀 테스트가 추가됩니다.
- fail-closed 동작이 removal 기반 drift와 attribute 기반 drift 모두에서 일관됩니다.
