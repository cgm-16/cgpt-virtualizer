## 배경

V1 whitepaper는 transcript unit을 `top-level matched bubble elements`로 고정합니다. 그런데 현재 구현은 transcript bubble을 subtree 전체에서 수집합니다.

## 근거

- `whitepaper.md:17`은 transcript unit을 top-level matched bubble로 정의합니다.
- `src/content/transcript-scan.ts:11-16`의 `collectTranscriptBubbles()`는 `root.querySelectorAll(bubbleSelector)`를 사용합니다.
- `src/content/rebuild.ts:160-179`의 dirty rebuild도 `selectors.transcriptRoot.querySelectorAll(selectors.bubbleSelector)`로 bubble을 다시 수집합니다.
- 현재 단위 테스트도 descendant-wide 동작만 고정합니다. `tests/unit/transcript-scan.test.ts:29-64`

## 문제

실제 selector가 bubble 내부의 중첩 descendant까지 매치하면 다음 문제가 생깁니다.

1. bubble 수를 잘못 세서 activation threshold 판단이 틀릴 수 있습니다.
2. nested node가 transcript unit처럼 측정되고 prefix sum에 포함됩니다.
3. patch/rebuild 과정에서 nested node를 transcript root의 direct child처럼 이동시켜 DOM 구조를 깨뜨릴 수 있습니다.

즉 현재 구현은 whitepaper의 top-level bubble 계약을 코드로 강제하지 않습니다.

## 기대 동작

- bubble 수집은 transcript root의 top-level bubble만 대상으로 제한되어야 합니다.
- dirty rebuild도 동일한 top-level 규칙으로 다시 수집해야 합니다.
- nested matched element가 있어도 transcript unit으로 취급되지 않아야 합니다.

## 완료 조건

- scan/rebuild 경로가 top-level bubble만 수집합니다.
- nested match가 있어도 bubble count, mounted range, rebuild 결과가 흔들리지 않습니다.
- top-level vs nested match를 구분하는 unit/integration 회귀 테스트가 추가됩니다.
