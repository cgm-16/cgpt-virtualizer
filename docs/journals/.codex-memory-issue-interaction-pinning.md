## 배경

whitepaper는 interaction safety를 V1 요구사항으로 명시합니다. 선택 중인 bubble, edit textarea가 열린 bubble, 필요하면 focused interactive control이 있는 bubble은 detach되면 안 됩니다.

## 근거

- `whitepaper.md:139-152`
- `src/content/state.ts:4-10`에는 `BubbleRecord.pinned` 필드가 있지만
- `src/content/state.ts:29-39`에서 항상 `false`로 초기화됩니다.
- `src/content/patch.ts:114-136`은 out-of-range record를 `pinned`와 무관하게 전부 detach합니다.
- `src/content` 아래에서 `pinned`, `Selection`, `textarea`, `activeElement`, `focus`를 실제로 처리하는 로직이 없습니다.

## 문제

현재 구현은 pin model의 데이터 필드만 있고 실제 pin 판정과 detachment 방지가 전혀 없습니다. 그래서 다음 상황에서 bubble이 가상화 대상이 되면 사용자 상호작용이 끊길 수 있습니다.

1. 텍스트를 드래그해서 선택 중인 bubble
2. edit textarea가 열린 bubble
3. focus가 걸린 interactive control이 있는 bubble

이 상태에서 scroll patch가 일어나면 whitepaper와 달리 해당 bubble이 detach될 수 있습니다.

## 기대 동작

- 선택 중인 bubble은 interaction이 끝날 때까지 detach되지 않아야 합니다.
- edit textarea mode가 열린 bubble은 detach되지 않아야 합니다.
- 필요하면 focused control도 pin 대상으로 포함해야 합니다.
- mounted range 계산 또는 patch 단계가 pin을 존중해야 합니다.

## 완료 조건

- pin 판정 로직이 추가됩니다.
- patch가 pinned bubble을 detach하지 않습니다.
- selection, textarea, focus 경로를 검증하는 회귀 테스트가 추가됩니다.
